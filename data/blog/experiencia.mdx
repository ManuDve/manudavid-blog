# JavaScript moderno

Aprender JavaScript moderno es una tarea sumamente dificil si no estuviste desde el comienzo. Es normal hoy en día ver ofertas de trabajo que incluyen distintas herramientas cuya rápida y constante evolución nos dificulta entender los problemas que buscan resolver. Aquellos valientes novatos que siguen la consigna de [leer la Documentación](/blog/el-mejor-curso), al buscar el porqué de la existencia de una herramienta, podrán encontrar jeroglíficos egicipcios del siguiente tipo:

> En esencia, webpack es un agrupador (bundler) estático de módulos para aplicaciones JavaScript modernas. Cuando webpack procesa tu aplicación, construye internamente un gráfico de dependencias desde uno o más puntos de entrada y luego combina cada módulo que tu proyecto necesita en uno o más paquetes, que son activos estáticos para servir tu contenido.\
> (Descripción del [Sitio de Webpack](https://webpack.js.org/concepts/))

Esta descripción, a ojos de alguien que recientemente aprendió la estructura básica de un sitio, simplemente no tiene sentido. Vamos al inicio.

## El comienzo

> "Aprende HTML, CSS y JavaScript" \
> (Sabio de la antigua China, fecha desconocida)

Sería un típico consejo para alguien que busca aprender a desarrollar sitios y aplicaciones web. Como muchos, así también comencé esta travesía hace más de una década. En aquel entonces los pasos —que están aún vigentes— eran bastante claros:

- Generar la estructura en HTML.
- Darle estilo con CSS.
- Agregar interactividad con JS.

```html #6
<!-- index.html -->
<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Mi sitio</title>
    <link rel="stylesheet" href="./css/style.css" />
    <script src="./js/main.js"></script>
  </head>
  <body>
    <h1>¡Hola desde HTML!</h1>
  </body>
</html>
```

```javascript
// main.js
console.log('¡Hola desde JavaScript!')
```

Digamos que ahora quiero agregar una funcionalidad mediante JavaScript a través de otro archivo en el HTML, puedo hacerlo agregando otra etiqueta en la cabecera como: `<script src="./js/otro-script.js"></script>`
